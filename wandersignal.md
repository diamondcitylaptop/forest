# Olive Resource Index

Olive Resource Index 是一个面向技术研究、知识工程和开源数据挖掘的轻量级外链资源聚合与导航系统。项目定位于为开发者、数据科学家和技术文档撰写者提供一套可本地化部署、可扩展、可版本化的外链资源管理方案，用于高效组织分散在多个数据源中的技术文档、分析报告、项目笔记和结构化数据文件。

本项目不依赖任何外部数据库或云服务，所有资源索引与元数据均以纯文本和 Markdown 格式存储于仓库中，支持离线查阅、全文检索和自动化脚本处理。Olive Resource Index 的目标用户包括但不限于：需要维护大量外链资源的技术团队、个人知识库管理者、开源项目文档维护者，以及需要对技术资料进行定期整理和审计的研发效能人员。

通过将散落在各个站点、仓库、笔记软件中的技术链接统一纳入一个轻量级的索引体系，Olive Resource Index 帮助用户减少链接失效、内容分散、难以追溯等常见问题，提升技术信息资产的长期可用性和可维护性。

## 功能概览

- **分级资源索引**：支持按主题、批次、来源等多维度对资源链接进行分层组织和标注，便于快速定位和筛选。

- **Markdown 原生呈现**：所有资源清单、目录结构和文档导航均以 Markdown 格式呈现，与主流代码托管平台和本地编辑器无缝兼容。

- **版本化外链管理**：通过 Git 版本控制系统记录每次资源增删改操作，实现外链集合的可追溯、可回滚和可审计。

- **自动化校验脚本**：内置基础链接可达性检查脚本，可定期扫描资源列表中的异常状态码或超时响应，辅助维护链接健康度。

- **结构化元数据扩展**：每个资源条目支持附加说明字段、标签分类和关联文档引用，方便构建轻量级知识图谱。

- **多场景导航视图**：提供按应用场景、技术领域和文档层级分类的多种导航表格，满足不同角色和任务的查阅习惯。

- **低门槛本地部署**：无需额外安装数据库或运行时环境，仅依赖标准 Git 和文本编辑器即可完成部署与日常使用。

- **开放贡献流程**：提供标准化的资源添加、更新和审核流程，便于团队协作和社区共建。

## 应用场景

- **技术文档团队的外链资产统一管理**：技术文档团队在撰写产品文档、API 参考或白皮书时，需要引用大量外部技术标准、开源项目地址和学术论文链接。Olive Resource Index 可作为统一的外链登记与审核面板，在文档发布前快速核对所有引用链接的有效性和版本信息。

- **开源项目维护者的参考资源归档**：开源项目维护者在日常开发中会积累大量相关的技术博客、议题讨论、第三方工具和示例代码仓库。通过 Olive Resource Index，可以将这些参考资源按模块、功能或版本进行分类归档，并随项目代码一同发布，降低新贡献者的学习门槛。

- **数据分析工程师的数据源索引管理**：数据分析工程师经常需要从多个公开数据平台、API 文档和数据集仓库获取数据。Olive Resource Index 支持记录数据源的访问方式、更新频率和字段说明，帮助团队成员快速理解数据依赖关系，减少因数据源变更导致的分析任务中断。

- **个人技术知识库的外链增强**：个人知识库作者在维护技术笔记或博客时，往往需要嵌入大量外部引用。Olive Resource Index 可作为知识库的外链附录，集中管理所有引用链接，并支持定期检查链接可用性，避免笔记内容因链接失效而贬值。

- **技术培训与教学资料的组织**：培训机构或高校教师在准备技术课程材料时，需要为学生提供大量的课外阅读链接和实验环境地址。Olive Resource Index 可按课程章节、难度等级或实验顺序组织这些链接，方便学生按需查阅，也便于教师每学期更新内容。

## 快速开始

以下步骤帮助您在本地环境中快速部署并运行 Olive Resource Index 基础功能。

```bash
# 1. 克隆项目仓库到本地
git clone https://github.com/zerxonhy/olive.git

# 2. 进入项目根目录
cd olive

# 3. 安装基础依赖（仅用于可选校验脚本，非强制）
# 若使用 Python 校验脚本，建议创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install requests

# 4. 运行基础链接可达性检查（示例脚本）
python scripts/check_links.py --source README.md

# 5. 通过本地编辑器或 Markdown 预览工具打开 README.md 查阅资源列表
```

## 贡献指南

我们欢迎并鼓励社区贡献者参与 Olive Resource Index 的资源扩充、脚本优化和文档改进。请遵循以下步骤提交贡献：

1. **创建 Issue 讨论**：在提交任何资源增删改之前，请先在 GitHub Issues 中创建一个新议题，说明您希望添加或修改的资源链接、分类建议以及理由。团队会在 2-3 个工作日内给予反馈。

2. **派生仓库并创建分支**：在获得初步确认后，请派生（Fork）主仓库到您的个人账户，并在派生仓库中创建一个功能分支，分支命名格式为 `feature/resource-{批次号}-{简述}` 或 `fix/link-{简述}`。

3. **按模板修改资源文件**：在 `resources/batch_75/` 目录下按照 `templates/resource_template.md` 的格式添加或更新资源条目。确保每个链接都附带简要说明、分类标签和来源备注。若涉及删除链接，请在 CHANGELOG.md 中注明失效原因或替换方案。

4. **运行本地校验脚本**：在提交前，请于本地环境执行 `scripts/check_links.py` 脚本，确保所有新增或修改的链接均可达且响应正常。若遇超时或错误，请检查链接格式或临时网络状况。

5. **发起 Pull Request**：将功能分支推送至您的派生仓库，然后向主仓库的 `main` 分支发起 Pull Request。请在 PR 描述中关联对应的 Issue 编号，并简要概述本次变更内容、测试结果和影响范围。团队会在收到 PR 后 5 个工作日内完成代码审查和合并。

## 常见问题

**Q: 链接检查脚本返回超时或连接错误，但浏览器中可以正常访问，如何解决？**

A: 该问题通常由目标站点的反爬机制或网络环境限制引起。建议首先检查本地网络代理设置，确保终端环境与浏览器使用相同的网络出口。其次，可在 `scripts/check_links.py` 中调整 `timeout` 参数（默认 10 秒）为更大值，例如 30 秒。此外，部分站点会拒绝 Python 默认的 User-Agent，可尝试在请求头中模拟常见浏览器的 User-Agent 字符串。

**Q: 如何批量导入大量现有链接，而不是逐条手动添加？**

A: 项目当前未提供一键式批量导入工具，但您可以通过编写简单脚本来转换现有链接列表。建议将链接按行存放于一个纯文本文件中，每行包含链接地址和可选注释（以制表符或逗号分隔），然后使用 Python 或 Shell 脚本读取该文件并按照 `resources/batch_*/index.json` 的格式生成对应的 JSON 条目。生成后请务必运行校验脚本确认所有链接有效性，再提交 Pull Request。

**Q: 资源列表中的链接发生永久迁移或内容变更，项目如何应对？**

A: Olive Resource Index 推荐贡献者定期通过链接检查脚本扫描全部资源，并关注 CHANGELOG.md 中记录的已知变更。若发现链接内容发生重大变化（如文档替换、域名迁移），建议在原条目中更新链接地址并保留旧地址作为备注（附加 `deprecated` 标签）。若链接彻底失效且无替代，应将该条目移至 `resources/archived/` 目录，并在索引中标记为失效，同时更新相关文档引用。
