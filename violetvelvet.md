# Olive Link Atlas

Olive Link Atlas 是一个面向开发者、技术研究人员与信息架构师的高质量外链与技术资源汇总系统。该项目并非简单的书签集合，而是一套具备结构化分类、可追溯源、版本受控的技术参考索引库。其目标用户包括正在探索新兴技术栈的工程师、需要维护团队知识库的技术负责人，以及希望从真实项目实践中提炼通用模式的研究者。

Olive Link Atlas 通过将分散于各处的深度技术文章、项目经验总结与架构设计方案进行集中编目与规范化描述，帮助用户显著降低信息筛选成本，提升技术决策效率。项目本身采用纯静态 Markdown 驱动，无需复杂后端环境，任何拥有 Git 工作流的开发者均可在一分钟内完成本地部署并开始使用。

## 功能概览

- **结构化资源编目**：所有收录的外链均按照技术领域、内容类型与成熟度等级进行多维标签化处理，支持快速过滤与定位。
- **版本化变更追踪**：基于 Git 原生能力记录每一条资源的添加、更新与移除历史，确保团队内对知识库的每一次修改均可审计。
- **全文元数据检索**：资源标题、摘要、标签与关联项目名称均纳入检索范围，支持通过命令行工具或编辑器插件进行即时查询。
- **自动化链接可用性检测**：内置基于 HTTP 状态码的定期检测机制，自动标记失效或响应超时的链接，辅助维护人员及时修复。
- **外部引用关系可视化**：自动分析各资源之间的相互引用情况，生成简单的依赖关系图谱，便于理解知识脉络。
- **多格式数据导出**：支持将当前资源列表导出为 JSON、YAML 或 CSV 格式，便于导入其他知识管理工具或用于自动化脚本处理。
- **可扩展的分类模板**：提供默认的分类体系模板，同时允许用户根据自身项目需要自定义新增分类与属性字段，无需修改核心代码。

## 应用场景

- **技术团队内部知识库建设**：技术负责人可以使用 Olive Link Atlas 统一收集团队在研发过程中发现的优质外部资料，避免知识碎片化。新成员入职时，可通过查阅该索引快速了解团队关注的技术方向与最佳实践来源。
- **开源项目文档的外部参考补充**：开源项目维护者常常需要在文档中引用外部规范、教程或相似实现。Olive Link Atlas 可作为这些外部引用的统一管理仓库，确保引用链接的长期可访问性，并在链接失效时能够迅速寻找替代来源。
- **技术调研与竞品分析**：在进行新技术选型或竞品功能对比时，研究人员可批量收集相关分析文章与案例研究，并通过项目的标签体系进行分类对比，显著提高调研报告的输出效率。
- **个人技术阅读流整理**：开发者可使用该项目维护个人技术阅读清单，按照学习主题规划阅读顺序，并记录每篇文章的核心结论与个人笔记，形成长期积累的学习资产。

## 快速开始

以下步骤将帮助您在本地环境中快速启动并运行 Olive Link Atlas。

```bash
# 1. 克隆项目仓库至本地
git clone https://github.com/zerxonhy/olive.git
cd olive

# 2. 安装项目依赖（项目基于 Node.js 构建，需要 npm 环境）
npm install

# 3. 执行本地构建与启动开发服务器
npm run build
npm start
```

执行上述命令后，项目将在本地端口 3000 上启动一个静态服务。您可以通过浏览器访问 `http://localhost:3000` 查看资源列表的主索引页面。若您仅需要使用命令行工具进行资源查询，可直接运行 `npm run query -- --tag=architecture` 进行测试。

## 贡献指南

我们欢迎并感谢任何形式的贡献，包括但不限于新增优质资源链接、改进现有资源描述、修复代码缺陷以及完善项目文档。为保证协作效率与质量，请遵循以下步骤：

1.  **查阅现有议题与拉取请求**：在提交新贡献之前，请先浏览项目的 Issues 列表与 Pull Requests，确认您计划处理的问题或建议尚未被他人认领或正在讨论中。若存在相关议题，请在下方留言表明您将着手处理，以避免重复工作。

2.  **从主分支创建功能分支**：所有代码与资源变更均应在独立的功能分支中进行。请从 `main` 分支检出新的分支，分支命名建议遵循 `feature/` 或 `fix/` 前缀，并简要描述变更内容，例如 `feature/add-kubernetes-resources`。

3.  **遵循项目的代码风格与提交规范**：JavaScript 代码请遵循 ESLint 配置（项目根目录包含 `.eslintrc.js`）。Commit 信息请采用约定式提交格式（Conventional Commits），即使用 `feat:`、`fix:`、`docs:`、`chore:` 等类型前缀，并附上清晰简洁的变更描述。

4.  **编写或更新相应的测试用例**：若您的变更涉及核心逻辑代码，请确保在 `tests/` 目录下新增或更新对应的单元测试，保证测试覆盖率达到现有水平。对于仅涉及资源 Markdown 文件的新增或修改，需运行 `npm run validate` 验证格式正确性。

5.  **提交拉取请求并等待代码审查**：完成变更后，将您的功能分支推送至远程仓库，并通过 GitHub 界面提交 Pull Request。请在 PR 描述中详细说明变更背景、实现方案以及测试结果。PR 将至少由一位项目维护者进行审查，审查通过后即可合并入主分支。

## 常见问题

**问：项目中的资源链接检测报告在哪里查看？检测到失效链接后应该如何操作？**

答：链接可用性检测结果默认输出至项目根目录下的 `reports/broken-links.json` 文件中。该报告会记录所有响应状态码非 2xx/3xx 的链接，并附带检测时间戳与 HTTP 状态码。您可以直接查看该文件，或通过 `npm run check` 命令在终端实时查看检测摘要。对于失效链接，我们建议首先访问原链接确认是否为临时性故障；若确认内容已永久迁移或移除，请在对应的资源 Markdown 文件中更新 URL 或移除该条目，并提交变更。

**问：我能否将 Olive Link Atlas 用于商业项目内部的知识库管理？是否需要支付任何费用？**

答：Olive Link Atlas 在 MIT 许可证下发布，这意味着您可以自由地将本项目用于任何商业或非商业目的，包括但不限于内部部署、修改、再分发以及作为其他产品的一部分。您无需支付任何授权费用，也无需公开您修改后的源代码，但请在分发时保留原始版权声明与许可证文本。

**问：项目支持从其他书签工具或知识库批量导入数据吗？**

答：目前项目原生支持导入基于特定 CSV 或 JSON 格式的数据文件。您可以将其他工具导出的数据按照 `data/import-schema.json` 中定义的格式进行转换，然后通过 `npm run import -- --file=your-data.json` 命令执行批量导入。对于浏览器书签（HTML 格式）的直接导入支持已在开发路线图中规划，但尚未在当前版本中提供。您可以参考 `docs/resource-management.md` 中的“批量操作”章节获取详细的导入示例与字段映射说明。
