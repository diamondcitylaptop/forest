# Olive Resource Aggregator

Olive Resource Aggregator 是一个面向开发者与系统架构师的轻量级技术资源外链汇总与导航工具。项目本身不托管具体内容，而是以结构化 Markdown 索引的方式，将分散在 GitHub 仓库中的技术文档、配置参考、实验笔记和架构方案集中呈现，便于团队或个人在本地搭建属于自己的技术知识导航站。

项目定位为“技术资源的资源”，适合需要频繁查阅外部技术文档、维护多份配置清单或跟踪多个实验性项目的用户。通过 Olive 提供的统一入口和分类索引，用户可以在不记忆大量原始链接的情况下，快速定位到所需的技术细节、信号配置、环境描述或实验记录。

## 功能概览

- **外链索引聚合** 提供统一的 Markdown 索引入口，将多个外部技术文档链接按主题分类，避免浏览器书签杂乱。
- **结构化目录导航** 每个索引条目均包含文档标题、简短描述和原始链接，支持人工查阅和脚本化解析。
- **本地静态部署** 项目本身为纯 Markdown 文件集合，无需数据库或后端服务，可直接通过 Git 克隆至本地，使用任意 Markdown 阅读器或静态站点生成器查看。
- **版本追踪友好** 所有索引变更通过 Git 记录，方便回溯文档更新历史或对比不同版本的配置差异。
- **可扩展分类模板** 提供统一的条目编写模板，用户可按需新增或调整分类，适应不同技术栈或项目阶段的需求。
- **快速检索支持** 索引文件采用一致的标题层级和标签约定，便于通过 grep 或 IDE 全局搜索快速找到目标条目。
- **轻量级协作** 支持多人通过 Pull Request 方式贡献新的资源链接，项目维护者可审核并合并，形成团队共享的知识索引。

## 应用场景

- **团队技术文档入口** 开发团队可将 Olive 作为内部 Wiki 的补充，集中存放外部依赖的配置说明、第三方库的用法参考或实验环境的搭建笔记，新成员加入时通过索引快速了解项目所依赖的外部资源。
- **个人实验笔记导航** 研究员或高级开发者在进行多个技术方案对比时，可将不同方案对应的配置文档、信号测试记录和环境描述通过 Olive 统一管理，避免在多个 GitHub 仓库间反复切换。
- **架构评审辅助材料** 架构师在设计评审前，可将涉及的外部参考文档（如中间件配置、网络策略示例、性能调优记录）汇总到 Olive，评审时直接展示索引结构，提高信息组织效率。
- **离线文档预备** 用户可在网络稳定时将 Olive 仓库完整克隆至本地，并结合其他工具将索引中的外链内容缓存为离线副本，适用于内网环境或移动办公场景。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，请确保已安装 Git 和任意 Markdown 阅读器。

```bash
# 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装依赖（项目本身无运行依赖，如需使用静态站点生成器，可自行安装）
# 例如安装 Python 和 MkDocs（可选）
# pip install mkdocs

# 本地预览索引内容
# 使用任意 Markdown 编辑器打开主索引文件
cat README.md

# 若使用 MkDocs，可启动本地预览服务
# mkdocs serve
```

## 贡献指南

1. **复刻项目并创建分支** 从主仓库复刻（Fork）至个人账号，然后基于 `main` 分支新建一个描述性的分支名称，例如 `add-new-signal-ref`。
2. **新增或更新索引条目** 在 `olive/` 目录下选择或创建对应的主题文件，按照 `templates/entry_template.md` 的格式添加新条目。每个条目需包含标题、简要说明和原始链接。请确保链接与用户原始数据一致，不添加额外前缀或改写。
3. **本地自查** 使用 Markdown 检查工具或编辑器预览渲染效果，确保表格、代码块和链接格式正确。若新增了分类，同步更新 `README.md` 中的功能概览或文档导航部分。
4. **提交并推送** 编写清晰的提交信息，例如 `docs: add signal configuration reference`，然后推送至个人复刻仓库。
5. **发起拉取请求** 向主仓库的 `main` 分支发起 Pull Request，在描述中说明新增或修改的条目列表及其用途。项目维护者将在两个工作日内审核，必要时提出修改意见，通过后合并。

## 常见问题

**Q：该项目是否需要运行任何服务或守护进程？**  
A：不需要。Olive 为纯静态 Markdown 索引，无需启动任何后台进程。用户只需通过 Git 克隆至本地，使用任意 Markdown 阅读器即可查阅全部内容。若希望获得更佳的浏览体验，可自行配置 MkDocs 或 VuePress 生成静态 HTML 站点。

**Q：外部链接失效或内容变更时，项目如何维护？**  
A：由于本项目仅索引外部链接，不托管实际内容，因此当用户发现链接失效或内容与描述不符时，可通过 GitHub Issue 报告。项目维护者会定期检查链接可达性，并在确认后更新对应条目或移除无效链接。用户亦可按贡献指南提交修正。

**Q：能否添加非 GitHub 域名的外部资源？**  
A：可以。项目定位为技术资源导航，不限制链接域名。但新增条目需遵循格式要求，包含清晰的标题、用途说明和原始 URL。原始 URL 必须按用户提供的形式原样写入，不得添加或修改协议前缀。
