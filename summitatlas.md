# Olive Atlas

Olive Atlas 是一个面向技术决策者、架构师与高级开发者的外链资源归集与结构化导航系统。本项目不提供代码库或运行时组件，而是以高度组织化的 Markdown 清单形式，收录了一批经过人工筛选的深度技术文档、架构分析报告与工程实践复盘。其核心定位是帮助用户在有限时间内快速定位到高价值的外部原始资料，减少信息检索中的噪声与重复劳动。

Olive Atlas 适用于以下场景：技术选型前需要进行竞品或方案对比、事故复盘阶段需要参考相似案例、架构设计评审前需要补充特定领域的背景知识、以及团队内部建立技术知识库时需要一套可维护的外部引用索引。本项目并不替代任何原始文档，而是作为入口层，提供清晰的上下文说明、标签分类与引用关系，使团队能够以标准化方式引用外部资源。

## 功能概览

- **结构化外链索引**：所有收录链接均按领域、成熟度与文档类型进行二级分类，支持快速过滤与批量导出。

- **文档元数据标注**：每个链接附带来源组织、最后更新时间推断、文档格式（如 Markdown、PDF、视频）与内容摘要，便于在不点击跳转前预判相关性。

- **多层级目录导航**：提供从宏观领域（如分布式系统、前端工程、云原生）到微观主题（如共识算法、构建工具插件、监控埋点）的渐进式浏览路径。

- **版本与快照引用**：对于 GitHub 仓库内的文档，标注具体的分支或提交哈希上下文，确保引用内容在时间上可追溯。

- **关联推荐图谱**：基于主题相似度与引用共现关系，在每条记录下方自动生成“参见”列表，辅助横向扩展阅读。

- **外部状态监控标记**：定期通过自动化流程检查链接可达性，并在清单中标记失效或重定向状态，降低维护成本。

- **自定义标签体系**：支持用户级标签追加，允许团队在私有分支中补充内部优先级、负责人或评审状态字段。

## 应用场景

- **架构评审准备**：当团队计划引入新的消息中间件或缓存方案时，架构师可快速查阅本项目中的“分布式一致性”与“高可用模式”分类，获取来自不同厂商与社区的实践对比报告，从而在评审会议中提供多维度论据。

- **事故复盘与根因分析**：在系统故障发生后，工程师可通过本项目的“事故案例”索引，找到类似故障的外部复盘文档，学习其他组织的应对策略与修复验证流程，避免重复踩坑。

- **技术培训与新人 onboarding**：团队可为新入职成员指定一份“必读清单”，直接引用本项目中的若干条外链作为学习材料，并配合元数据中的难度等级与预估阅读时间，制定合理的学习计划。

- **技术雷达与趋势追踪**：技术负责人可定期浏览本项目定期更新的“新兴技术”分类，了解服务网格、WebAssembly、FinOps 等领域的最新动态，为季度技术规划提供信息输入。

- **开源合规审查辅助**：法务或合规人员可通过本项目中的许可证与版权声明索引，快速跳转至各开源组件的原始授权文件，加速合规清单编制。

## 快速开始

以下命令可在任意 POSIX 兼容环境中克隆本项目的导航仓库，并安装本地预览工具（基于 Node.js 的静态站点生成器），最后启动开发服务器以便在浏览器中浏览完整的资源索引。

```bash
# 克隆仓库
git clone https://github.com/zerxonhy/olive.git olive-atlas
cd olive-atlas

# 安装依赖（使用 npm 或 yarn）
npm install

# 生成静态索引页面并启动本地服务
npm run build
npm start
```

启动成功后，访问控制台输出的本地地址（通常为 <code>http://localhost:3000</code>）即可查看资源导航首页。若只需查看原始 Markdown 清单文件，可直接进入 <code>./docs/</code> 目录，使用任何 Markdown 阅读器打开对应的分类文件。

## 贡献指南

我们欢迎并鼓励社区贡献，包括新增高质量外链、更新失效链接、补充元数据信息以及改进分类体系。请遵循以下步骤以确保变更顺利合并。

1.  **查阅现有分类与标签定义**：在添加新链接前，请先浏览 <code>./docs/reference/taxonomy.md</code> 确认是否存在匹配的分类与标签。若无合适分类，可在提交中附带建议新增的分类名称及理由。

2.  **克隆仓库并创建特性分支**：使用 <code>git checkout -b feature/add-new-resource</code> 创建独立分支，避免直接在主分支上操作。

3.  **修改对应分类文件并更新元数据**：在 <code>./docs/categories/</code> 下找到合适的 Markdown 文件，按照 <code>./docs/reference/citation-format.md</code> 中规定的格式追加链接条目，并完整填写标题、来源组织、文档类型、摘要及标签字段。

4.  **运行本地验证脚本**：在提交前执行 <code>npm run validate-links</code> 检查所有链接（含新增）的可达性，并执行 <code>npm run build</code> 确保静态站点生成无错误。若验证失败，请修复后再提交。

5.  **提交 Pull Request**：推送分支到远程仓库，并在 GitHub 上创建 Pull Request。请在描述中说明新增资源的背景与价值，以及是否涉及分类调整。项目维护者将在两个工作日内进行评审。

## 常见问题

**问：本项目是否提供实际的可运行代码或库文件？**

答：不提供。Olive Atlas 是一个纯粹的外链索引与导航项目，所有内容均为 Markdown 格式的清单与元数据描述。它不包含任何可执行程序、SDK、API 实现或二进制文件。用户需要自行访问链接指向的原始仓库或文档以获取具体技术内容。

**问：收录链接的标准是什么？是否会收录个人博客或未经评审的文档？**

答：本项目的收录标准以内容的技术深度、结构完整性和来源可追溯性为首要考量。优先收录来自知名组织、会议演讲、期刊预印本或具有明确版本控制的仓库文档。个人博客仅在内容被多个独立来源引用且经过至少一位维护者全文审读后才会考虑。本项目不保证收录内容的正确性或时效性，用户应自行判断。

**问：如果发现某个链接失效或内容已迁移，应该如何处理？**

答：请按照贡献指南提交 Issue 或 Pull Request。在提交时，若能够找到新地址，请同时提供新旧两个 URL，并在元数据中注明“迁移”状态。若无法找到替代地址，维护者会将该条目标记为“失效”并从活跃索引中移出，但仍保留在历史归档中以便追溯。
