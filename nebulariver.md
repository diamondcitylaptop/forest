# Olive Link Hub

Olive Link Hub 是一个面向技术研究者、开源贡献者和信息聚合需求者的轻量化外链资源导航系统。该项目定位于对分散在多个仓库、文档和社区中的高价值技术外链进行结构化整理与可回溯管理，帮助用户在不增加复杂依赖的前提下，快速建立个人或团队的知识外链索引。Olive Link Hub 本身不存储实体文件，而是通过严格的 Markdown 清单体系，将分散的 GitHub 技术笔记、配置方案和实验记录聚合为统一的检索入口。

目标用户包括频繁查阅技术 Spec 的开发者、需要维护团队知识库的架构师，以及希望从零开始构建个人外链树的开源爱好者。项目遵循“最小可用”原则，所有数据均以纯文本 Markdown 承载，无需数据库或后端服务，可直接部署为静态站点或作为子模块嵌入现有文档体系。

## 功能概览

- **外链结构化登记** 提供标准化的链接条目模板，支持分类、标签和优先级标记，每条链接可附加单行摘要说明。
- **批次化导入管理** 按项目批次（如第 31/107 批）组织链接集合，便于追溯新增来源和更新周期。
- **状态与有效性检查** 内置链接存活状态的手动记录字段，支持标记失效、待验证或长期有效状态。
- **多维度检索支持** 基于 grep 友好的目录结构，可按文件名、关键词或提交哈希快速定位目标外链。
- **可嵌入的只读视图** 生成的链接清单可直接复制到其他 README 或 Wiki 中，保持格式一致性。
- **离线访问友好** 所有资源列表均为静态文本，无需联网即可浏览已有外链条目。
- **扩展字段预留** 每一条目预留注释区域，可用于记录访问时间、快照摘要或替代地址。
- **变更历史追踪** 配合 Git 原生日志，可清晰查看每次链接增删改的提交记录与作者信息。

## 应用场景

- **技术文档外链附录管理** 团队在编写项目文档时，常需引用外部 Spec、教程或工具站。Olive Link Hub 可作为统一的附录仓库，集中维护所有外部引用，避免文档内散落裸链接。
- **开源项目依赖说明索引** 开源项目的依赖库、参考实现或衍生作品往往分布在多个仓库。使用 Olive Link Hub 可整理这些关联仓库的地址与用途说明，降低新贡献者的理解成本。
- **个人知识库的“书签”系统** 个人笔记中常保存大量网页链接。Olive Link Hub 提供轻量级结构化方案，以 Markdown 列表配合分类目录，替代浏览器书签栏的杂乱堆叠。
- **离线环境下的资源清单传递** 在内部网络受限的环境中，可通过 Olive Link Hub 导出完整链接清单，配合简单的文本工具即可实现资源地址的批量传递与核对。
- **多批次增量式收藏** 对于持续更新的资源集合（如每期技术周报），可按批次编号分别记录，保留每次收录的时间上下文，方便后续回顾新增趋势。

## 快速开始

以下步骤指导您在本机克隆并运行 Olive Link Hub 的基础查看环境。项目本身为静态 Markdown 集合，无需编译，但提供可选的本地预览服务。

```bash
# 1. 克隆仓库
git clone https://github.com/zerxonhy/olive-hub.git
cd olive-hub

# 2. 安装轻量级 Markdown 预览工具（以 live-server 或 grip 为例）
npm install -g live-server

# 3. 启动预览服务，访问当前目录下的 README 及链接清单
live-server --port=8080 --entry-file=README.md
```

若无需预览服务，直接使用任意 Markdown 编辑器打开 `README.md` 或各批次清单文件即可浏览全部外链。

## 贡献指南

Olive Link Hub 欢迎外部贡献者提交新的链接批次、更新失效地址或改进模板规范。请遵循以下步骤以确保合并顺畅。

1. **Fork 本仓库** 并在您的 Fork 中创建一个新的分支，分支名建议使用 `feat/batch-NNN` 或 `fix/link-status` 格式，便于识别变更目的。
2. **遵循批次模板** 在 `batches/` 目录下新增或修改批次文件。每条链接必须包含原始 URL（使用 <code> 包裹）、简短摘要以及状态标记（如 `有效`、`待验证`、`失效`）。若为新增批次，请同步更新 `batches/index.md` 中的索引。
3. **本地自检** 运行可选脚本 `scripts/check_links.sh`（若已配置）验证链接格式是否符合规范。同时确保所有 URL 按原样复制，不添加额外前缀或修改域名大小写。
4. **提交变更并推送到您的 Fork**，随后在本仓库发起 Pull Request。请在 PR 描述中说明本批次链接的来源背景或更新原因，便于维护者评估。
5. **等待维护者审核** 若链接内容与项目定位一致且格式无误，维护者将合并 PR。合并后，对应批次即成为正式索引的一部分。

## 常见问题

**Q: 我能否直接编辑 `batches/` 下的现有批次文件来修改某个链接地址？**

A: 可以。但请注意，若修改的是已合并的历史批次，建议在 PR 中额外说明修改原因（如源仓库迁移、链接失效更换为 archive 地址等）。为避免混淆，对于失效链接，优先推荐将其状态标记为“失效”并新增一个条目指向替代地址，而不是直接覆盖原有 URL。

**Q: 如果我想引入其他来源的链接（不在 olive 仓库内），应该怎么做？**

A: 完全允许。Olive Link Hub 定位为通用外链索引，不限制来源。您只需按照 `templates/link_template.md` 的格式在对应批次中新增条目，并在摘要中注明来源站点或原始上下文即可。若链接数量较多，建议单独开启一个新的批次文件。
