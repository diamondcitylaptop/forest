# Olive Link Atlas

Olive Link Atlas 是一个面向开发者、技术研究员与开源生态观察者的高密度外链资源汇总与结构化导航系统。该项目不提供代码托管或在线服务，而是以静态文档仓库的形式，对分散在互联网各处的深度技术文章、项目经验复盘、架构设计笔记与工程实践文档进行人工筛选、分类索引与关联标注。其目标用户包括需要快速获取某一技术领域高质量外链的技术决策者、正在调研开源方案的产品经理，以及希望从真实项目中学习工程化写法的开发者。Olive Link Atlas 通过严格的条目格式、上下文标签与摘要说明，帮助用户避免在低质量内容中耗费时间，将阅读转化效率提升至可用级别。

## 功能概览

- **多维度外链分类体系**：按技术领域、应用场景、作者背景与文档类型四个维度对每一条外链进行标签化处理，支持用户按自身需求快速定位。
- **条目级摘要与价值标注**：每条收录链接均附带一段由项目维护者撰写的中文摘要，说明该文档解决了什么问题、适合什么阶段的技术人员阅读。
- **关联文档网状推荐**：基于内容主题和引用关系，在相邻条目之间建立“延伸阅读”链接，帮助用户在单个技术话题下形成连续阅读路径。
- **批次化更新与版本记录**：项目以批次为单位组织新增链接，每批次均记录添加日期、审核人与变更摘要，便于追踪内容演进。
- **纯 Markdown 静态交付**：全部内容以 Markdown 文件形式存储于 GitHub 仓库，无需数据库或后端服务，用户可随时 fork 或 clone 到本地进行二次整理。
- **社区外链提名与审核流程**：允许用户通过 Issue 提交推荐链接，经维护团队审核通过后纳入后续批次，形成开放但有序的内容共建机制。
- **全文检索友好型目录结构**：文件名与标题采用英文小写与连字符命名规范，配合目录索引文件，支持 GitHub 原生搜索以及本地 grep 高效检索。

## 应用场景

1. **技术选型调研**：当团队需要评估某一技术栈或中间件时，可直接查阅本项目中对应分类下的外链，阅读来自真实生产环境的经验总结，而非仅依赖官方宣传文档。
2. **架构设计复盘参考**：在进行系统重构或新项目架构设计前，通过本项目的“架构与设计”分类获取同类系统的设计笔记与踩坑记录，减少设计决策中的盲区。
3. **开源贡献前准备**：计划向某开源项目提交 PR 或参与社区讨论前，利用本项目汇总的相关贡献指南与维护者沟通记录，快速了解社区风格与审查标准。
4. **技术写作素材收集**：技术博主或文档工程师可通过本项目按主题浏览大量真实技术文档，获取写作灵感、结构案例与术语表达参考。
5. **离线知识库构建**：通过 git clone 将整个仓库同步至本地，结合个人笔记工具或静态站点生成器，构建属于自己的技术外链知识库。

## 快速开始

以下命令可在任意支持 git 与 node.js 的环境中完成本项目的完整克隆与本地预览。本项目不依赖外部服务，所有操作均在本地完成。

```bash
# 克隆仓库到本地
git clone https://github.com/zerxonhy/olive.git

# 进入项目根目录
cd olive

# 安装用于本地预览的静态服务器依赖（可选）
npm install -g serve

# 启动本地预览服务，默认监听端口 3000
serve -p 3000
```

## 贡献指南

本项目以开放但有序的方式接受外部贡献。所有贡献者需遵守以下步骤，以确保条目质量和索引一致性。

1. **阅读并遵守条目规范**：在提交新外链或修改现有摘要前，务必阅读 `docs/guides/editing.md` 中的摘要编写规范，确保摘要长度、语气与信息密度符合项目标准。
2. **通过 Issue 提交新链接提名**：使用 `.github/ISSUE_TEMPLATE/` 中提供的提名模板创建 Issue，填写原始 URL、推荐分类、个人摘要草稿以及推荐理由。维护团队将在 5 个工作日内进行初审。
3. **等待审核与分类确认**：审核通过后，维护者会为 Issue 添加 `approved` 标签并分配批次编号。此时贡献者无需自行修改仓库文件。
4. **在指定批次内提交 Pull Request**：维护者将向贡献者发送包含目标批次目录的 PR 邀请。贡献者需在该 PR 中按模板格式添加新条目文件，并确保所有链接无格式错误。
5. **签署开发者原创声明**：首次贡献者需在 PR 评论中明确声明所提交摘要为本人原创或已获得授权转载，否则该 PR 将无法合并。

## 常见问题

**Q: 本项目与普通书签收藏夹或网络导航站有何本质区别？**

A: 普通收藏夹仅保存 URL，缺乏上下文与质量筛选。Olive Link Atlas 的核心价值在于人工编写的结构化摘要、分类标签以及批次化审核流程。每条外链均经过至少一位维护者阅读并提炼核心要点，而不是简单收录链接。此外，关联推荐机制帮助用户从一条链接出发，沿着技术脉络发现更多相关文档，形成系统化的阅读路径。

**Q: 为什么所有链接都指向同一个 GitHub 仓库下的不同 Markdown 文件？**

A: 这是项目的设计选择。Olive Link Atlas 本身是一个“关于技术文档的技术文档”，所有被收录的外链都以 Markdown 文件形式存在于 olive 仓库中，便于版本控制、内容审查和离线同步。这些文件中的内容可能本身包含对外部网络资源的二次引用，但项目只负责本仓库内的文档索引，不对外部链接的可用性负责。

**Q: 我想将本项目用于商业课程或内部培训材料，是否需要额外授权？**

A: 本项目整体采用 MIT 许可证，您可以在遵守许可证条款的前提下自由使用、复制、修改和分发。但请注意，本项目收录的每一条外链指向的原始文档分别属于各自的作者或组织，其版权与使用限制以原始文档所声明的许可证为准。本项目仅提供索引和摘要，不主张对原始文档的任何权利。
