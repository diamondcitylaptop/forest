# Horizon Olive

Horizon Olive 是一个面向开发者与技术研究团队的外链资源整合与结构化导航系统。该项目以纯静态 Markdown 与元数据驱动的方式，将分散于网络各处的技术文档、工具站点、社区讨论与实验性仓库进行统一索引与分类管理。其核心目标是解决技术信息过载与资源失联问题，帮助用户在特定技术栈或研究领域内快速定位高质量的外部参考材料。

Horizon Olive 适用于技术团队内部知识库建设、开源项目依赖链追踪、以及个人开发者对新兴技术方向的系统性调研。通过将资源链接与语义标签、状态标记和关联说明进行绑定，本项目不仅充当外链收藏夹，更提供轻量级的资源生命周期管理能力，包括有效性检测、版本关联记录与使用场景标注。

## 功能概览

- **外链分类索引**：按技术领域、资源类型与成熟度对每条外部链接进行多维度分类，支持快速筛选与排序。

- **资源状态标注**：为每个资源条目标记活跃度、维护状态、兼容性范围与最后验证时间，便于判断信息时效性。

- **关联关系图谱**：通过统一命名与引用标记，展示不同资源之间的依赖、衍生或对比关系，形成网状知识结构。

- **内容摘要提取**：每条资源均附带人工撰写或自动生成的内容摘要，说明其核心用途、适用场景与注意事项。

- **版本与快照记录**：对关键外部链接记录其内容变更的版本节点，支持回溯历史形态，降低依赖漂移风险。

- **本地预览服务**：提供轻量级 HTTP 服务器，支持在本地或内网环境以网页形式浏览资源目录与详情页。

- **元数据导出接口**：支持将全部资源索引导出为 JSON 或 CSV 格式，便于导入下游分析工具或文档生成系统。

## 应用场景

- **技术选型调研**：团队在评估数据库中间件或前端框架时，可通过 Horizon Olive 快速访问对比文章、性能测试报告与社区讨论链接，减少重复搜索成本。

- **项目依赖文档整理**：开源项目维护者可利用本系统记录上游依赖的文档地址、问题跟踪页面与发布公告，确保依赖变更时可追溯原始信息源。

- **新人培训路径规划**：技术负责人可将入门教程、最佳实践与常见问题解答等外链按学习阶段组织成目录，帮助新成员系统性地掌握必要知识。

- **离线文档镜像参考**：在网络受限环境中，团队可提前将资源列表导出并配合本地缓存工具，保留关键外链的参考快照。

- **技术周报素材聚合**：技术编辑或社区管理者可定期将推荐阅读的文章、仓库与视频链接录入系统，生成结构化周报草稿。

## 快速开始

以下步骤将指导您在本地环境中克隆、安装依赖并启动 Horizon Olive 预览服务。

```bash
# 克隆仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装依赖（基于 Node.js 环境，使用 npm）
npm install

# 启动本地预览服务，默认监听端口 8080
npm run serve
```

启动后，在浏览器中访问 `http://localhost:8080` 即可浏览资源索引页面。如需修改端口或绑定主机地址，请参考 `config/local.yaml` 配置文件。

5. 推送分支至您的 Fork，并在 GitHub 上向本仓库的主分支发起 Pull Request。请在 PR 描述中清晰说明变更目的、影响范围以及是否涉及破坏性改动。

## 常见问题

**问：启动预览服务后无法加载任何资源列表，页面显示空白，如何排查？**

答：请检查 `data/index.json` 文件是否存在且内容格式正确。首次运行时会自动从 `src/core/parser.js` 中定义的默认分类生成初始索引。若文件损坏，可删除 `data/index.json` 后重启服务，系统将重建索引。同时请确认 `config/local.yaml` 中的资源目录路径配置正确。

**问：某些外部链接失效或内容已变更，项目能否自动检测？**

答：项目提供 `npm run validate` 脚本，可对索引中所有外链发送 HEAD 请求并检查响应状态码。失效链接会记录到 `logs/broken-links.log` 文件中。您可配置定时任务（如 cron）定期执行该脚本，并将日志输出至监控系统。目前版本不支持自动修复或更新内容，需人工介入处理。

**问：是否支持多用户协同编辑资源列表？**

答：本项目当前设计为单机或内网部署模式，不内置用户认证与冲突合并机制。多用户协同可通过版本控制系统（如 Git）间接实现，每个贡献者通过分支和 Pull Request 提交变更，由项目维护者审核合并。若需实时协同编辑，建议配合第三方 WebDAV 或在线文档平台，并将最终变更同步回仓库。
