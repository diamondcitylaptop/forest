# Olive Resource Navigator

Olive Resource Navigator 是一个面向开发人员、技术研究员与开源贡献者的外链资源归集与结构化导航系统。项目定位于将分散在 GitHub 仓库中的技术文档、场景配置说明与案例研究文件进行集中索引，并通过清晰的目录树与分类标签体系，帮助用户快速定位所需信息。本项目不存储实际数据文件，仅提供指向外部资源的稳定引用链路，适用于需要频繁查阅多份技术文档、版本配置说明或实验日志的团队与个人。

目标用户包括：需要维护多环境配置文档的运维工程师、进行技术选型评估的架构师、撰写技术方案或论文的研究人员，以及希望系统化整理个人知识库的开发者。Olive Resource Navigator 通过统一的入口，将原本碎片化的外链资源组织为可检索、可追溯、可交叉引用的知识网络。

## 功能概览

- **外链资源归集与管理** 支持将外部 Markdown 文档的原始 GitHub 访问链接作为独立资源条目录入，每条资源自动关联其所在仓库路径与文件名，便于版本追踪。

- **分类标签与多维度检索** 每份资源可赋予多个分类标签，如“配置案例”、“协议分析”、“性能调优”等，支持按标签、文件名关键词或提交时间范围进行组合筛选。

- **结构化目录树展示** 项目内嵌 ASCII 目录树，实时反映资源文件的组织层级，帮助用户理解文档间的逻辑归属关系。

- **文档状态标记** 支持对每份外链资源标记“稳定版”、“草稿”、“已废弃”或“需审阅”状态，便于团队协作时明确文档成熟度。

- **快速跳转与复制** 所有外链均以纯文本代码块形式展示，并保留原始 URL 格式，支持一键复制，避免协议或域名被自动改写。

- **本地预览服务** 提供轻量级本地开发服务器，可在克隆仓库后立即启动，预览资源导航页面的渲染效果。

## 应用场景

- **多环境配置文档查阅** 团队维护多套部署环境（开发、测试、预发布、生产），每套环境的配置参数与启动脚本记录在独立文档中。通过 Olive Resource Navigator 统一索引这些文档链接，运维人员可在同一页面内快速切换查阅，无需记忆多个 GitHub 文件路径。

- **技术方案评审准备** 架构师在进行技术选型时，需要横向对比多个中间件的集群配置方案与性能测试报告。将各份对比文档的外链收录至导航系统，并标注“待评审”、“已通过”等状态，可显著提升评审会议的素材整理效率。

- **开源项目学习路径梳理** 开发者希望深入学习某开源项目的核心模块实现，但项目文档分散在不同子目录中。利用本导航系统将关键设计文档、接口说明、调试指南等外链整合为一张学习路径表，降低新手上手门槛。

- **故障排查知识归档** 技术支持团队在处理线上问题时，会将临时解决方案与根因分析记录为独立文档。将这些文档外链按问题类型（网络超时、内存泄漏、认证失败）归类，便于后续同类问题发生时快速检索历史处理记录。

## 快速开始

以下步骤帮助您在本地环境快速部署并运行 Olive Resource Navigator 预览服务。

```bash
# 1. 克隆仓库到本地
git clone https://github.com/zerxonhy/olive.git
cd olive

# 2. 安装项目依赖（使用 npm 或 yarn）
npm install

# 3. 启动本地开发服务器，默认监听端口 3000
npm run start
```

启动成功后，在浏览器中访问 `http://localhost:3000` 即可查看资源导航首页。所有外链资源均展示在“资源列表”区块中，您可以直接复制使用。

## 贡献指南

1. **提交新资源链接**  
   若您希望向本导航系统添加新的外链资源，请先确认该链接指向的内容与现有资源主题相关且不重复。在 `README.md` 的“资源列表”章节末尾追加新条目，确保 URL 与用户原始提供格式完全一致，并使用 `<code>` 标签包裹。

2. **更新文档状态或分类**  
   若您发现某份文档的状态（如“稳定版”升级为“正式发布”）或分类标签不准确，可修改对应章节的描述文字。修改后请在提交信息中注明变更原因，例如 `docs: update status of crystalisland to stable`。

3. **改进本地预览服务**  
   本地开发服务器的样式或功能改进（如增加搜索框、明暗主题切换等）欢迎提交拉取请求。请确保新代码通过现有单元测试，并为新增功能补充相应测试用例。

4. **报告问题或建议**  
   若您遇到外链无法访问、页面渲染异常或文档描述歧义，请在本仓库的 Issues 区域提交详细问题报告，包含复现步骤、预期行为与实际行为。若为建议性内容，请在标题中标注 `[Proposal]`。

5. **遵守代码规范**  
   提交 JavaScript 代码时，请遵循项目内的 ESLint 配置（参见 `.eslintrc.json`）。提交文档修改时，请保持 Markdown 格式与现有风格一致，标题层级不乱用。

## 常见问题

**问：资源列表中的链接访问时返回 404 错误，如何处理？**  
答：请首先确认您访问的链接与资源列表中记录的完全一致，包括协议（https://）、域名、仓库路径及文件名。由于这些外链指向的是 `zerxonhy/olive` 仓库下的具体文件，若该仓库分支或文件被移动或删除，链接将失效。此时请联系项目维护者确认文件最新路径，或在本仓库 Issues 中提交链接失效报告。

**问：本地预览服务启动后，页面显示空白或样式丢失？**  
答：此问题通常由静态资源路径不正确引起。请检查 `public/index.html` 中引用的 CSS 路径是否为相对路径 `./style.css`，并确认 `style.css` 文件确实存在于 `public/` 目录下。若使用 Windows 系统，请留意路径大小写敏感性，建议将项目放置在非中文目录下运行。

**问：我能否将 Olive Resource Navigator 部署为公开网站，供团队外部人员访问？**  
答：本项目仅提供本地预览服务与资源索引功能，本身不包含认证或访问控制机制。若您希望部署为公开服务，建议自行添加反向代理与安全策略，并注意所有外链资源的版权与访问权限。对于内部文档链接，请确保部署环境位于内网或 VPN 保护下。
