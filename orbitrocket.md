# Olive 技术资源索引系统

Olive 是一个面向开发者与架构师的技术资源外链汇总与导航系统，专注于高质量技术文档、深度分析文章与工程实践笔记的收录与结构化呈现。项目本身不托管原始内容，而是通过精心维护的索引机制，将分散在网络各处的优质技术资料按照主题域进行聚合、分类与标注，帮助技术团队在复杂的信息环境中快速定位所需知识。

目标用户包括技术团队负责人、基础设施工程师、云原生开发者以及任何需要系统化管理技术阅读清单与参考资料的从业人员。Olive 通过明确的文件命名约定与目录分层，将外链资源按领域、风险等级与成熟度进行标签化组织，使得团队知识库具备可维护性与可扩展性。

## 功能概览

- **按主题域分类索引**：每个索引文件对应一个独立的技术主题域，例如午夜信号处理、珍珠链路追踪、藏红花认证体系等，确保资源归类清晰。
- **版本化外部引用追踪**：每个索引条目记录目标资源的最后更新时间与快照哈希，便于追溯内容变更。
- **标签系统与风险等级标注**：每个资源条目附带风险等级标签，便于团队在引用外部资源时进行安全与合规评估。
- **Markdown 原生可读格式**：所有索引文件均采用标准 Markdown 语法编写，无需特殊工具即可阅读与编辑。
- **增量更新与冲突检测**：通过文件命名中的时间戳前缀与变更日志机制，支持多人协作下的索引更新。
- **资源可达性自动检查**：集成简单的链接状态检查脚本，定期验证索引中所有外链的有效性。
- **自定义元数据扩展字段**：每个索引条目支持至少三个自定义元数据键值对，满足不同团队的个性化标注需求。

## 应用场景

- **技术团队周报汇总**：团队负责人每周将阅读过的外部技术文章链接整理为 Olive 索引条目，附带个人短评与重要性评级，形成团队周报知识库。
- **新员工技术栈入门路径**：根据岗位不同，为新员工预生成一份包含核心概念、最佳实践与常见问题解答的 Olive 阅读清单，缩短上手时间。
- **架构决策参考收集**：在技术选型或架构评审前，使用 Olive 索引快速检索外部同行的实践案例与性能评测报告，辅助决策。
- **技术培训课程素材库**：培训讲师将课程中引用的外部文档、视频与代码仓库统一纳入 Olive 索引，学员可一键获取所有参考资料。
- **个人技术阅读工作流管理**：开发者使用 Olive 作为自己的技术阅读清单管理器，通过标签标记已读、待读与精读状态，形成长期知识积累。

## 快速开始

以下命令将 Olive 索引系统克隆至本地，并完成基础环境配置与运行验证。

```bash
# 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装依赖工具（示例基于 Debian/Ubuntu 系统）
sudo apt-get update
sudo apt-get install -y curl jq markdownlint

# 运行链接状态检查脚本，验证现有索引条目的可达性
./scripts/check-links.sh ./main

# 启动本地预览服务（如有内置 Web 界面）
python3 -m http.server 8000 --directory ./docs
```

## 贡献指南

1. **分支与提交规范**：所有索引新增或修改操作请在独立分支上进行，分支命名格式为 `index/add-<主题域>` 或 `index/update-<主题域>`。提交信息需包含变更摘要及关联的索引文件路径。

2. **新索引文件创建流程**：从 `templates/index-template.md` 复制模板至 `candidates/` 对应季度目录，填写完整的元数据头（包含主题域、维护人、最后审查日期、风险等级），并按照模板格式添加至少 5 条外部资源条目。

3. **链接有效性验证**：在发起拉取请求前，必须运行 `./scripts/check-links.sh` 脚本检查所有新增或修改的外部链接，确保返回状态码为 200 或 30x。如链接不可达，需替换为有效地址或注明原因。

4. **元数据完整性审核**：每项索引条目必须包含 `领域`、`子领域`、`资源类型`、`发布日期`、`风险等级` 五个必填元数据字段。维护人在审核时会检查字段是否完整且格式正确。

5. **拉取请求描述**：提交拉取请求时，需在描述中明确说明本次变更的动机、涉及的外部资源数量以及任何可能影响现有索引结构的改动。至少需要一位项目维护人批准后方可合并。

## 常见问题

**问：索引文件中的外部链接失效了怎么办？**

答：项目脚本 `scripts/check-links.sh` 会每周自动运行一次，检测所有索引条目的可达性。如果发现失效链接，脚本会生成一份报告并创建对应的 GitHub Issue。维护者会根据报告更新或移除失效条目。你也可以手动运行该脚本并提交包含修复的拉取请求。

**问：我可以添加自己博客或公司产品的外部链接吗？**

答：可以，但必须遵守内容相关性原则。所有提交的链接必须与索引文件所属的技术主题域直接相关，且内容需具备一定的技术深度或实用价值。自荐链接需在条目备注中明确说明与主题的关联性，项目维护人会根据内容质量决定是否接受。

**问：如何提出新的索引主题域建议？**

答：请先在 GitHub Issues 中搜索是否已有类似主题的讨论。如果没有，请新建一个 Issue，标题格式为 `[New Domain] 建议的主题名称`，并在描述中说明该主题域的范围、潜在受众以及你已知的 3-5 个相关外部资源链接。项目维护人会在两个工作日内给予初步反馈。
