# Olive Bridge

Olive Bridge 是一个面向技术文档与开源知识库的轻量级外链汇总与结构化导航系统。它并非传统的搜索引擎或网页存档工具，而是一个专注于将分散的技术资源、规范文档、项目笔记与社区讨论进行语义化归类与快速索引的中间层平台。项目主要服务于需要频繁查阅大量跨项目文档的开发者、技术撰稿人以及开源社区维护者，帮助其从杂乱的链接收藏中解脱出来，建立清晰、可共享、可版本控制的技术知识图谱。

Olive Bridge 的核心定位是“技术资源的链接中枢”。它不存储具体内容，而是通过严谨的元数据描述与分类体系，将外部有价值的 Markdown 文档、规范说明、项目蓝图等资源组织成有序的导航结构。用户可通过本项目快速定位到所需的技术决策记录、架构设计说明或模块接口定义，极大提升在大型代码库或复杂技术体系中的信息检索效率。

## 功能概览

- **语义化资源归类**：依据技术领域、文档类型、项目阶段等多维度标签对导入的链接进行自动与手动分类，支持构建层次化的导航目录。

- **结构化元数据提取**：自动解析目标 Markdown 文档的标题、一级标题树、代码块语言分布等元信息，生成资源摘要卡片，便于预览。

- **外链状态健康检查**：定期对收录的 URL 进行可访问性探测与内容哈希比对，标记失效链接或内容变更，确保导航库的时效性。

- **版本化导航快照**：与 Git 版本控制系统深度集成，每次导航结构或资源列表的变更均生成可追溯的提交记录，支持回溯任意历史状态。

- **自定义视图模板**：提供多种预置的导航页面布局（目录式、看板式、时间线式），用户可根据使用习惯选择或定制展示方式。

- **协作批注与评论**：允许项目成员对特定资源链接附加技术批注、使用心得或勘误提醒，形成群体智慧沉淀。

- **快速模糊检索**：基于资源标题、描述、分类标签及批注内容构建轻量级倒排索引，支持毫秒级的关键词与过滤条件组合检索。

## 应用场景

- **大型微服务架构文档导航**：在包含数十个微服务模块的项目中，各服务的设计文档、API 定义、部署说明分散在不同仓库。团队可使用 Olive Bridge 统一收录这些文档链接，并按服务名称或功能域分类，新成员可快速了解服务全貌。

- **技术规范与决策记录管理**：技术委员会或架构组可将历次技术选型对比、架构评审纪要、编码规范等外部链接录入系统，附加评审结论与适用条件批注。后续相同技术决策场景可直接参考历史记录。

- **开源社区贡献者入门向导**：开源项目维护者可将贡献指南、行为准则、新手任务列表、开发环境搭建教程等关键资源通过 Olive Bridge 组织为阶梯式学习路径，降低社区新人的参与门槛。

- **技术培训与知识库构建**：企业技术培训部门可将内部课程资料、实验手册、视频录像链接按课程阶段和技能等级归类，形成可复用的培训资源库，支持不同岗位员工按需学习。

- **多项目依赖关系追踪**：对于存在上下游依赖的多个项目，可将各自的项目说明、接口变更日志、版本发布计划等链接汇总于一处，清晰展示项目间的依赖图谱与信息同步节点。

## 快速开始

以下步骤将帮助您在本地环境中快速启动 Olive Bridge 服务。

```bash
# 1. 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git

# 2. 进入项目根目录
cd olive

# 3. 安装核心依赖（使用 npm，需 Node.js 18+）
npm install

# 4. 复制环境变量配置文件并填充必要参数
cp .env.example .env

# 5. 初始化本地元数据库（使用 SQLite）
npm run db:init

# 6. 启动开发服务器
npm run dev
```

启动成功后，访问控制台输出的本地地址（默认为 `http://localhost:3000`）即可开始使用 Olive Bridge 的导航管理界面。

## 常见问题

**问：Olive Bridge 是否存储所收录链接的原始文件内容或快照？**

答：不存储。Olive Bridge 仅存储用户显式提供的资源标题、描述、分类标签以及通过解析器从目标文档中提取的轻量级元数据（如标题层级、代码块语言统计）。系统不缓存或代理原始文件内容，所有内容访问均重定向至原始链接。链接有效性检查仅通过 HEAD 请求验证状态码，不会下载完整文件体。

**问：如何将 Olive Bridge 中已分类的资源列表共享给团队其他成员？**

答：由于所有导航结构、分类标签和资源元数据均以 SQLite 数据库文件或版本化的配置文件形式存储在项目目录中，您只需将整个项目仓库（或指定的数据库文件与配置目录）通过 Git 仓库、共享存储或压缩包方式分发给团队成员。成员拉取后，通过运行 `npm run db:migrate` 同步数据库结构，即可获得完全一致的导航视图。也支持通过内置的 `npm run export:snapshot` 命令导出为纯 JSON 或 YAML 格式的静态配置文件。

**问：系统提示“链接验证失败”，但浏览器中可正常访问该 URL，是什么原因？**

答：这通常由以下原因导致：
1.  目标服务器禁止了 HEAD 请求方法，或对非浏览器 User-Agent 返回非标准状态码。
2.  网络环境存在防火墙或代理策略，导致服务端出口 IP 被限制。
3.  链接为需要特定 Cookie 或动态 JavaScript 渲染才能正常响应的页面。
解决方案：您可在环境变量中配置 `VALIDATOR_USE_GET=true` 以改用 GET 请求进行验证（会增加流量消耗）；或将该链接的验证策略配置为“忽略”模式，仅依靠人工定期检查。对于高频访问的内部文档站点，建议在站点侧将 Olive Bridge 服务 IP 加入白名单。
