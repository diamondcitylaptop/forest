# Olive Link Atlas

Olive Link Atlas 是一个面向开发者、技术研究员与开源项目维护者的高质量外链聚合与导航系统。本项目不提供内容托管，而是通过精心筛选与分类的技术资源链接，帮助用户快速定位到优质的外部文档、技术笔记、项目案例与社区讨论。项目定位为“技术外链的精选目录”，目标用户包括正在寻找特定技术实现参考的工程师、需要追踪多个仓库变更的集成负责人，以及希望从已有开源实践中学习架构设计的初学者。

本项目核心价值在于将散落在各个仓库角落的 Markdown 文档、配置示例与技术心得进行集中索引，并按照主题领域、成熟度与维护状态进行标注，从而降低信息检索成本。Olive Link Atlas 自身不存储业务数据，所有记录均以只读方式引用外部公开资源，确保项目轻量化且易于分叉定制。

## 功能概览

- **按领域分类的链接索引**：所有收录链接按照技术栈、业务场景或文档类型进行一级分类，支持快速过滤与检索。

- **多仓库关联映射**：针对同一组织或同一技术体系下的多个仓库文档，提供关联视图，便于理解项目全貌。

- **文档状态标记**：对每个链接指向的文档标注维护状态（活跃、归档、实验性），辅助用户判断信息时效性。

- **简易集成接口**：提供 JSON 与 YAML 格式的链接清单导出，方便其他工具或脚本批量处理。

- **变更追踪记录**：项目维护者可通过更新日志追溯每次链接增删改的原因，保持索引透明度。

- **自定义标签体系**：允许用户通过 Pull Request 或 Issue 建议新增标签分类，社区驱动分类演进。

- **轻量级部署**：纯静态 Markdown 生成，无需数据库，兼容 GitHub Pages 或任何静态托管服务。

## 应用场景

- **新成员入职技术栈速查**：团队新加入的开发者可通过本项目的资源列表快速了解团队常用工具链、内部文档位置以及关键决策记录，减少入门阶段的摸索时间。

- **技术选型前的调研辅助**：架构师在进行中间件或框架选型时，可通过本项目汇总的对比分析链接、性能测试报告链接以及社区讨论链接，快速获取多维度信息，而不必在多个仓库间手动翻找。

- **开源项目维护者的外部依赖索引**：当一个开源项目依赖多个上游仓库或参考多个外部设计文档时，维护者可将这些外部链接统一收录于 Olive Link Atlas，确保团队成员使用一致的参考版本，避免引用混乱。

- **离线文档镜像的源列表**：对于需要在内网环境搭建文档镜像的团队，本项目提供的结构化链接列表可直接作为爬虫或同步脚本的输入源，保证镜像内容与上游保持同步。

- **技术博客或教程的参考文献库**：技术写作者在撰写系列文章时，可将每篇文章涉及的外部参考链接集中存放于本项目，便于读者延伸阅读，同时也方便作者跨文章维护链接一致性。

## 快速开始

以下步骤帮助您在本地环境中快速克隆、安装依赖并运行 Olive Link Atlas 的静态站点生成流程。

```bash
# 1. 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 2. 安装依赖（假设使用 Node.js 生态）
npm install

# 3. 生成静态站点（输出至 ./dist 目录）
npm run build

# 4. 启动本地预览服务（默认端口 8080）
npm run serve
```

执行完上述命令后，打开浏览器访问 `http://localhost:8080` 即可查看本地生成的导航页面。若需更新链接列表，请直接编辑 `./data/links.yml` 文件后重新运行构建命令。

## 贡献指南

我们欢迎并感谢任何形式的贡献，包括但不限于新增链接建议、分类优化、文档改进以及 Bug 报告。请遵循以下步骤：

1. **查阅现有链接**：在提交新链接建议前，请通过 `./data/links.yml` 文件确认该链接尚未被收录，避免重复。若链接已存在但分类不当，请优先提出分类调整 Issue。

2. **创建 Issue 讨论**：对于新增链接或重大分类调整，请先在 GitHub Issue 中创建讨论帖，说明链接来源、所属领域及推荐理由，等待维护者反馈后再进行实际操作。

3. **分叉仓库并提交变更**：获得初步共识后，请分叉本项目到您的个人账户，在分叉中编辑 `./data/links.yml` 文件，并按照现有格式填写链接 URL、标题、分类标签及简短描述。若涉及新增分类，需同步更新 `categories.yml`。

4. **运行本地验证**：提交 Pull Request 前，请在本地执行 `npm run validate` 确保所有链接可访问且元数据格式正确。验证通过后，执行 `npm run build` 确保构建无报错。

5. **发起 Pull Request**：将您的变更以 Pull Request 形式提交回本项目，并在描述中引用对应的 Issue 编号。维护者将在 3 个工作日内审核，审核通过后合并进入主分支，并自动触发站点重新部署。

## 常见问题

**Q：本项目是否会对收录链接进行内容缓存或备份？**

A：不会。Olive Link Atlas 仅存储 URL 及其元数据，不缓存任何外部文档的内容。所有链接均以原始形式引用，用户点击后将直接跳转至源仓库或源页面。我们建议用户自行关注源文档的更新与可用性。

**Q：如果发现某个收录链接已经失效或内容大幅变更，应如何处理？**

A：您可以通过 GitHub Issue 提交链接失效报告，或直接按照贡献指南提交 Pull Request 修改或移除该链接。维护者会定期运行自动化链接检查脚本，但社区报告仍然是保持索引质量的重要途径。

**Q：是否可以请求将私有仓库或内网链接收录至本项目？**

A：本项目仅收录可公开访问的链接。私有仓库或内网地址不在收录范围内，因为其内容无法被普遍访问，且可能涉及访问权限问题。对于公开但需要登录凭据的链接，同样不予收录。
